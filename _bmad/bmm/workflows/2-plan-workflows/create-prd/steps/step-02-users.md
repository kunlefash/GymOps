# Step 02: Deep-Dive User Personas

## Instructions
1. Expand on the personas from the product brief
2. Create detailed persona cards with demographics, behaviors, and motivations
3. Map user journeys for each persona

<ask>
For each persona identified in the brief:
- What is their role/title and experience level?
- What are their primary goals when using this product?
- What frustrations do they have with current solutions?
- What is their technical proficiency?
- How frequently would they use this product?
</ask>

<action>
For each persona, create a detailed persona card:
- Name and role
- Demographics and context
- Goals and motivations
- Pain points and frustrations
- Current workflow and tools
- Success criteria from their perspective
</action>

<template-output file="_bmad-output/planning-artifacts/prd-{name}.md" section="user-personas">
Document each persona card in detail.
Include a user journey map showing key touchpoints and pain points.
</template-output>
