# Step 09: Definition of Done Gate

## Instructions
Final quality gate before pushing code.

<check>
## Definition of Done
- [ ] All acceptance criteria met (verify each one)
- [ ] All tasks and subtasks complete
- [ ] All tests passing (unit + integration + E2E)
- [ ] Test coverage ≥ 80% on new/modified files
- [ ] Zero linting errors
- [ ] Zero TypeScript errors
- [ ] No console.log statements left in production code
- [ ] No hardcoded secrets or credentials
- [ ] Code follows project conventions (conventional commits, file structure)
- [ ] Story file updated with dev agent record
</check>

If ANY DoD check fails, fix before proceeding to step 10.
If all checks pass, proceed to push and PR.
