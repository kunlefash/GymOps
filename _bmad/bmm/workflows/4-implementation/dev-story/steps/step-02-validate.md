# Step 02: Validate Story Structure

## Instructions
Ensure the story has everything needed for development.

<check>
- [ ] Story has at least one task with subtasks
- [ ] Each task has a clear description
- [ ] Acceptance criteria are present and testable
- [ ] File list is populated
- [ ] Story points assigned
- [ ] No unresolved dependencies
</check>

If validation fails, attempt to fill gaps from PRD/architecture context.
If critical info is missing, halt and request clarification.
