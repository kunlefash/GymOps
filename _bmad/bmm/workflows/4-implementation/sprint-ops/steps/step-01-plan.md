# Step 01: Sprint Planning

## Instructions
1. Load all stories from `_bmad-output/implementation-artifacts/stories/`
2. Load sprint status template
3. Plan the sprint based on priority, dependencies, and capacity

<ask>
Sprint planning time!

1. Sprint number? (or "next")
2. Sprint duration? (1 week / 2 weeks)
3. Start date?
4. Sprint goal? (one sentence)
5. How many story points can we take on? (velocity or estimate)
</ask>

<action>
1. Gather all "ready-for-dev" stories
2. Sort by priority and dependencies
3. Select stories that fit within velocity
4. Create sprint file from template: `_bmad-output/implementation-artifacts/sprints/sprint-{N}.yaml`
5. Update selected stories' status to "sprint-planned"
</action>

<template-output file="_bmad-output/implementation-artifacts/sprints/sprint-{N}.yaml">
Sprint configuration with selected stories, goals, and timeline.
</template-output>
